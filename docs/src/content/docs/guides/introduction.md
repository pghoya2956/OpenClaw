## OpenClaw이란

OpenClaw은 **AI 에이전트 플랫폼**이다. Slack, Discord, Telegram 등 다양한 메시징 채널에 AI 에이전트를 연결하고, 각 에이전트가 독립적인 워크스페이스에서 도구와 스킬을 사용하여 자율적으로 작업을 수행한다.

핵심 특징:

- **멀티채널**: 하나의 게이트웨이가 여러 메시징 플랫폼을 동시에 관리
- **멀티에이전트**: 독립적인 에이전트가 각자의 세션, 스킬, 도구를 보유
- **피어 아키텍처**: 에이전트 간에 부모-자식 관계가 없는 동등한 피어
- **설정 기반**: 코드 변경 없이 YAML/JSON 설정만으로 에이전트를 정의하고 라우팅

## 이 가이드의 목적

[공식 문서](https://docs.openclaw.ai)는 **사용자 가이드**로, "어떻게 설정하고 사용하는가"에 초점이 있다. 설정 예제, CLI 명령어, 채널별 셋업 절차를 다룬다.

이 가이드는 **소스 코드 레벨의 학습 가이드**로, "내부에서 어떻게 동작하는가"에 초점이 있다:

| 공식 문서 | 이 가이드 |
|----------|----------|
| Slack 채널 설정 방법 | Slack 메시지가 도착해서 응답이 전송되기까지의 코드 흐름 |
| 에이전트 설정 옵션 | 에이전트 런타임이 LLM을 호출하고 도구를 실행하는 루프 구조 |
| 세션 리셋 설정 | 세션 데이터의 JSONL 저장 형식과 컴팩션 알고리즘 |
| 스킬 설치 방법 | 스킬이 파싱되어 시스템 프롬프트에 주입되는 과정 |

## 코드베이스 규모

OpenClaw 소스 코드(`openclaw/src/`)의 규모:

- **4,800개+** TypeScript 소스 파일
- **483,000줄** 이상의 코드
- 주요 모듈: gateway, channels, agents, routing, sessions, skills, tools, plugins, config

이 가이드는 이 방대한 코드베이스에서 **메시지 흐름을 따라가는 순서**로 핵심 모듈을 하나씩 풀어간다.

## 대상 독자

- OpenClaw을 운영하면서 내부 동작을 이해하고 싶은 운영자
- OpenClaw에 기능을 추가하거나 커스터마이징하려는 개발자
- AI 에이전트 플랫폼의 아키텍처 설계를 참고하려는 엔지니어

## 학습 순서

이 가이드는 **메시지가 시스템을 관통하는 흐름**을 따라가도록 설계되었다:

```
메시지 도착 → 채널 수신 → 라우팅 → 에이전트 실행 → 응답 전송
```

각 섹션은 이 흐름의 한 단계를 깊이 다룬다:

- **아키텍처 개요** — 전체 컴포넌트와 관계를 조감도로 파악
- **메시지 흐름** — Slack 메시지 하나가 시스템 전체를 관통하는 여정
- **설정 시스템** — 모든 동작의 기반이 되는 설정 로딩 파이프라인
- **게이트웨이** — 컨트롤 플레인 역할의 서버 구조
- **채널 시스템** — 외부 메시징 플랫폼과의 연결 추상화
- **라우팅** — 메시지를 적절한 에이전트로 전달하는 매칭 알고리즘
- **에이전트 런타임** — LLM 호출과 도구 실행의 루프 구조
- **세션 관리** — 대화 상태의 영속화와 최적화
- **스킬/도구/플러그인** — 에이전트 기능 확장 포인트

## 소스 코드 탐색 팁

이 가이드에서 소스 코드를 참조할 때는 항상 `openclaw/src/` 기준의 상대 경로를 사용한다:

```
openclaw/src/gateway/server.impl.ts    → 게이트웨이 서버 진입점
openclaw/src/routing/resolve-route.ts  → 라우팅 알고리즘
openclaw/src/agents/pi-embedded-runner/run.ts → 에이전트 실행 루프
```

코드를 직접 따라가고 싶다면, 이 가이드의 각 섹션에서 언급하는 파일을 에디터에서 열고 함께 읽는 것을 권장한다.
