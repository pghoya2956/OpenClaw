## 개요

OpenClaw은 설정 파일이 변경되면 게이트웨이를 재시작하지 않고도 변경사항을 적용할 수 있다. 이를 **핫 리로드**라고 한다.

**핵심 파일**: `gateway/config-reload.ts`, `gateway/server-reload-handlers.ts`

## 동작 원리

### 변경 감지

`startGatewayConfigReloader()` 함수가 설정 파일을 주기적으로 감시한다:

```
설정 파일 해시 계산
→ 이전 해시와 비교
→ 변경 감지 시 리로드 트리거
```

설정 파일의 해시값(`ConfigFileSnapshot.hash`)을 비교하여 내용이 실제로 변경되었는지 확인한다.

### 리로드 프로세스

변경이 감지되면 `createGatewayReloadHandlers()`에 등록된 핸들러들이 순차적으로 실행된다:

```
새 설정 파일 읽기 → readConfigFileSnapshot()
→ 유효성 검증
→ 검증 실패 시: 경고 로그 + 기존 설정 유지
→ 검증 성공 시: 각 서브시스템에 새 설정 전파
```

### SIGUSR1 시그널

`SIGUSR1` 시그널을 수신하면 게이트웨이가 재시작된다. 이는 핫 리로드가 아닌 프로세스 재시작이며, `setGatewaySigusr1RestartPolicy()`에서 정책이 설정된다.

```bash
# SIGUSR1로 게이트웨이 재시작
kill -USR1 <gateway-pid>
```

## 리로드 가능한 설정

모든 설정이 핫 리로드를 지원하는 것은 아니다:

| 설정 | 핫 리로드 | 비고 |
|------|----------|------|
| 채널 설정 | 지원 | 채널 모니터 재시작 |
| 에이전트 설정 | 지원 | 새 요청부터 적용 |
| 스킬 설정 | 지원 | 스킬 워처가 파일 변경 감시 |
| 모델 설정 | 지원 | 새 요청부터 적용 |
| 게이트웨이 포트 | 미지원 | 프로세스 재시작 필요 |
| TLS 설정 | 미지원 | 프로세스 재시작 필요 |
| 플러그인 목록 | 미지원 | 프로세스 재시작 필요 |

## 스킬 핫 리로드

스킬은 별도의 파일 워처(`agents/skills/refresh.ts`)가 감시한다. `registerSkillsChangeListener()` 함수가 워크스페이스의 `skills/` 디렉토리를 감시하며, `SKILL.md` 파일이 변경되면 해당 에이전트의 스킬 목록을 다시 로딩한다.

```
SKILL.md 파일 변경 감지
→ 해당 에이전트의 스킬 재로딩
→ 다음 에이전트 실행부터 새 스킬 적용
→ 진행 중인 실행에는 영향 없음
```

## 안전장치

핫 리로드 시 유효하지 않은 설정은 거부된다:

- Zod 검증 실패 → 기존 설정 유지 + 경고 로그
- 파일 읽기 실패 → 기존 설정 유지 + 에러 로그
- 부분 업데이트 불가 → 전체 설정을 한 번에 교체
