## 개요

리셋 정책은 세션을 자동으로 초기화하는 규칙이다. 오래된 컨텍스트가 누적되어 에이전트의 응답 품질이 저하되는 것을 방지한다.

**핵심 파일**: `config/sessions/reset.ts`

## 리셋 모드

### daily (일별 리셋)

매일 지정된 시각에 세션을 리셋한다:

```yaml
session:
  reset:
    mode: "daily"
    atHour: 6          # 매일 오전 6시에 리셋
```

마지막 활동 시각이 오늘의 리셋 시각 이전이면, 다음 메시지에서 새 세션을 시작한다.

### idle (유휴 리셋)

일정 시간 동안 활동이 없으면 리셋한다:

```yaml
session:
  reset:
    mode: "idle"
    idleMinutes: 120    # 2시간 유휴 후 리셋
```

마지막 메시지로부터 지정된 시간이 경과하면, 다음 메시지에서 새 세션을 시작한다.

## 대화 유형별 리셋

`resetByType`으로 대화 유형별 다른 정책을 적용할 수 있다:

```yaml
session:
  resetByType:
    direct:                # DM
      mode: "idle"
      idleMinutes: 120
    group:                 # 그룹 대화
      mode: "daily"
      atHour: 0
    thread:                # 스레드
      mode: "idle"
      idleMinutes: 60
```

## 리셋 동작

세션이 리셋되면:

```
리셋 조건 충족 감지
→ 기존 세션 메타데이터 보존 (sessions.json에 기록)
→ 기존 트랜스크립트 보존 (JSONL 파일 유지)
→ 새 세션 ID 생성
→ 빈 히스토리로 새 세션 시작
→ 다음 메시지부터 새 세션에 기록
```

리셋은 기존 데이터를 **삭제**하지 않는다. 새 세션을 시작할 뿐이며, 이전 트랜스크립트는 그대로 남아 있어 나중에 조회할 수 있다.

## 수동 리셋

`/reset` 슬래시 커맨드로 수동 리셋:

```
/reset
→ 현재 세션 즉시 리셋
→ 새 세션 시작
```

RPC 메서드 `sessions.reset`으로도 가능하다.

## 리셋 감지

에이전트가 실행될 때 세션 리셋 여부를 확인한다:

```
세션 키로 기존 세션 조회
→ 리셋 정책 평가
→ 리셋 필요 시: 새 세션으로 실행
→ 리셋 불필요 시: 기존 세션에 히스토리 추가
```
